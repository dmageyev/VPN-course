# Глосарій — Лекція 07 — Сучасний протокол WireGuard та концепція Zero Trust (ZTNA)

| Термін / Скорочення | Визначення |
|---------------------|-----------|
| WireGuard | Сучасний VPN-протокол (~4000 рядків коду), вбудований у ядро Linux 5.6+. Відрізняється простотою, швидкістю та сучасними криптопримітивами. |
| Curve25519 | Еліптична крива для ECDH у WireGuard. Забезпечує обмін ключами Diffie-Hellman зі 128-бітною стійкістю. |
| BLAKE2s | Криптографічна хеш-функція, що використовується у WireGuard для хешування та KDF. Швидша за SHA-2 при порівняній безпеці. |
| Noise Protocol Framework | Криптографічний фреймворк, на якому побудований WireGuard. Визначає безпечне рукостискання та управління ключами. |
| Cryptokey Routing | Концепція WireGuard: кожному публічному ключу відповідають дозволені IP-діапазони. Спрощує маршрутизацію та авторизацію. |
| Roaming (WireGuard) | Здатність WireGuard-клієнта змінювати IP-адресу без переузгодження тунелю. Реалізовано через відстеження останньої активної endpoint-адреси. |
| Zero Trust Network Access (ZTNA) | Концепція безпеки «нульової довіри»: кожен запит автентифікується та авторизується незалежно від розташування (всередині чи ззовні мережі). |
| Zero Trust | Принцип безпеки: «Ніколи не довіряй, завжди перевіряй». Мережева позиція не є основою для надання доступу. |
| SDP (Software-Defined Perimeter) | Архітектура Zero Trust, де захищений периметр будується динамічно навколо кожного користувача/ресурсу, а не навколо мережі. |
| Identity-Aware Proxy (IAP) | Проксі, що перевіряє ідентичність та контекст перед наданням доступу до ресурсу. Ключовий компонент ZTNA-архітектур. |
| BeyondCorp | Модель Zero Trust від Google, що реалізує ZTNA для всіх корпоративних застосунків незалежно від мережевого розташування. |
| Tailscale | VPN-сервіс на основі WireGuard з централізованою панеллю керування (control plane). Реалізує mesh-топологію зі DERP-релеями. |
| Headscale | Відкрита self-hosted альтернатива control plane Tailscale для управління WireGuard mesh-мережею. |
| MagicDNS | Функція Tailscale: автоматичне DNS-розв'язання імен вузлів у mesh-мережі без ручного налаштування DNS. |
| DERP (Designated Encrypted Relay for Packets) | Протокол та сервери-ретранслятори Tailscale для передачі WireGuard-трафіку через NAT у разі, коли пряме підключення неможливе. |
| Continuous Authentication | Безперервна автентифікація у Zero Trust: ідентичність підтверджується повторно при кожному запиті або через короткі інтервали. |
| Мікросегментація | Техніка Zero Trust: поділ мережі на ізольовані мікросегменти з окремими політиками доступу для кожного сервісу або застосунку. |
| Key rotation (WireGuard) | Регулярна ротація попередньо розподілених ключів (PSK) у WireGuard для підвищення PFS та захисту від майбутнього квантового зламу. |
| Preshared Key (PSK) | Додатковий симетричний ключ у WireGuard поверх ECDH. Підвищує квантову стійкість та забезпечує додатковий шар безпеки. |
| AllowedIPs | Параметр WireGuard peer: список IP-діапазонів, які маршрутизуються через цього peer. Основа Cryptokey Routing. |
