# Глосарій — Лекція 04 — Протоколи тунелювання

| Термін / Скорочення | Визначення |
|---------------------|-----------|
| GRE (Generic Routing Encapsulation) | Протокол тунелювання (RFC 2784) для інкапсуляції пакетів будь-якого протоколу у IP. Не шифрує, часто поєднується з IPsec. |
| PPTP (Point-to-Point Tunneling Protocol) | Застарілий VPN-протокол Microsoft (RFC 2637) на основі PPP. Вважається небезпечним через вразливості MS-CHAPv2 та MPPE. |
| L2TP (Layer 2 Tunneling Protocol) | Протокол тунелювання канального рівня (RFC 2661). Не шифрує самостійно; зазвичай використовується разом з IPsec (L2TP/IPsec). |
| L2TP/IPsec | Комбінація L2TP для тунелювання та IPsec для шифрування. Поширений у корпоративних VPN, вбудований у Windows, macOS, iOS, Android. |
| SSTP (Secure Socket Tunneling Protocol) | VPN-протокол Microsoft (RFC 5246), що тунелює PPP через TLS/HTTPS (порт 443). Ефективно обходить більшість файрволів. |
| IKEv2 (Internet Key Exchange version 2) | Протокол обміну ключами для IPsec (RFC 7296). Підтримує MOBIKE, швидке перепідключення, EAP-автентифікацію. |
| ESP (Encapsulating Security Payload) | Протокол IPsec (RFC 4303), що забезпечує шифрування, автентифікацію та захист від replay-атак. Протокол IP 50. |
| AH (Authentication Header) | Протокол IPsec (RFC 4302), що забезпечує тільки автентифікацію та цілісність без шифрування. Протокол IP 51. |
| Тунельний режим (Tunnel Mode) | Режим IPsec, де шифрується весь вихідний IP-пакет включно із заголовком; додається новий заголовок. Для Site-to-Site VPN. |
| Транспортний режим (Transport Mode) | Режим IPsec, де шифрується лише корисне навантаження; вихідний IP-заголовок зберігається. Для Host-to-Host захисту. |
| SSTP | Secure Socket Tunneling Protocol — VPN на базі SSL/TLS, порт 443. Вбудований у Windows, ефективно проходить NAT та файрволи. |
| PPP (Point-to-Point Protocol) | Протокол канального рівня для послідовних з'єднань. Основа PPTP та L2TP; забезпечує IP-адресацію та автентифікацію (PAP, CHAP). |
| MS-CHAPv2 | Microsoft Challenge Handshake Authentication Protocol v2 — протокол автентифікації. Вразливий до офлайн-атак перебором; не рекомендується без захисного тунеля. |
| NAT Traversal (NAT-T) | Механізм для роботи IPsec через NAT шляхом інкапсуляції ESP у UDP (порт 4500). |
| Протокол-пасажир (Passenger Protocol) | У тунелюванні — вихідний пакет, що інкапсулюється (IPv4, IPv6, Ethernet). |
| Протокол-носій (Carrier Protocol) | У тунелюванні — зовнішній протокол, у який вкладається пакет-пасажир (GRE, IPsec, VXLAN). |
| OpenVPN | VPN-рішення з відкритим кодом, що використовує TLS для управління та шифрування трафіку. Підтримує UDP/TCP, порт 1194 за замовчуванням. |
| WireGuard | Сучасний мінімалістичний VPN-протокол (~4000 рядків коду), вбудований у ядро Linux 5.6+. Використовує Curve25519, ChaCha20-Poly1305, BLAKE2s. |
| Split Tunneling | Режим VPN, де лише частина трафіку направляється через тунель, а решта — напряму в Інтернет. |
| Full Tunneling | Режим VPN, де весь трафік клієнта направляється через VPN-тунель. Забезпечує максимальний контроль. |
