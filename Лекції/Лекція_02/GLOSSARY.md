# Глосарій — Лекція 02 — Асиметричне шифрування, цифрові підписи, PKI та центри сертифікації

| Термін / Скорочення | Визначення |
|---------------------|-----------|
| RSA (Rivest–Shamir–Adleman) | Асиметричний алгоритм шифрування та електронного підпису, що базується на складності факторизації великих чисел. Розміри ключів: 2048–4096 біт. |
| ECC (Elliptic Curve Cryptography) | Асиметрична криптографія на основі еліптичних кривих. Забезпечує еквівалентну стійкість при значно менших розмірах ключів порівняно з RSA. |
| ECDH (Elliptic Curve Diffie-Hellman) | Протокол обміну ключами на основі еліптичних кривих. Основа для Perfect Forward Secrecy у TLS та IKE. |
| Diffie-Hellman (DH) | Протокол обміну ключами у відкритому каналі без попереднього спільного секрету. Основа більшості VPN-протоколів для узгодження ключів. |
| PKI (Public Key Infrastructure) | Інфраструктура відкритих ключів — набір ролей, політик і процедур для управління цифровими сертифікатами та ключами шифрування. |
| CA (Certificate Authority) | Центр сертифікації — довірена організація, що видає та підписує цифрові сертифікати X.509. |
| X.509 | Стандарт формату цифрових сертифікатів. Сертифікат X.509 містить відкритий ключ, ідентифікатор власника, підпис CA та термін дії. |
| CSR (Certificate Signing Request) | Запит на підписання сертифіката — файл, що містить відкритий ключ та інформацію про суб'єкт, надсилається в CA для отримання сертифіката. |
| CRL (Certificate Revocation List) | Список відкликаних сертифікатів — список серійних номерів сертифікатів, які CA визнав недійсними до закінчення терміну дії. |
| OCSP (Online Certificate Status Protocol) | Протокол онлайн-перевірки статусу сертифіката в реальному часі — альтернатива CRL. |
| TLS (Transport Layer Security) | Криптографічний протокол захисту транспортного рівня. Забезпечує конфіденційність, цілісність та автентифікацію у мережевій передачі. |
| PFS (Perfect Forward Secrecy) | Властивість протоколу: компрометація довгострокового ключа не розкриває сесійних ключів минулих сеансів. Реалізується через ECDH. |
| ECDSA (Elliptic Curve Digital Signature Algorithm) | Алгоритм цифрового підпису на основі еліптичних кривих. Використовується у TLS-сертифікатах та SSH-ключах. |
| Цифровий підпис | Криптографічний механізм підтвердження автентичності та цілісності повідомлення за допомогою приватного ключа відправника. |
| Root CA | Кореневий центр сертифікації — верхній рівень ієрархії PKI, чий сертифікат є самопідписаним та попередньо встановлений у довірені сховища. |
| Intermediate CA | Проміжний центр сертифікації — підпорядкований Root CA. Видає кінцеві сертифікати, ізолюючи Root CA від прямих операцій. |
| Let's Encrypt | Безкоштовний автоматизований CA з підтримкою протоколу ACME для автоматичного отримання та оновлення TLS-сертифікатів. |
| ACME (Automatic Certificate Management Environment) | Протокол автоматизації отримання та оновлення сертифікатів TLS, розроблений IETF і використовуваний Let's Encrypt. |
| Curve25519 | Еліптична крива, оптимізована для ECDH. Використовується у WireGuard та сучасних реалізаціях TLS завдяки швидкості та безпеці. |
| Ed25519 | Схема цифрового підпису на основі кривої Edwards25519. Використовується у SSH, WireGuard та PGP як стандарт сучасного підпису. |
| PKCS#12 (.pfx/.p12) | Бінарний формат контейнера для зберігання сертифіката та приватного ключа разом, захищений паролем. |
| PEM (Privacy-Enhanced Mail) | Текстовий формат кодування сертифікатів та ключів у Base64 між заголовками BEGIN/END. Найпоширеніший формат у Linux/OpenSSL. |
