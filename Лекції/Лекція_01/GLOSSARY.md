# Глосарій — Лекція 01 — Основи кібербезпеки та введення в технології VPN

| Термін / Скорочення | Визначення |
|---------------------|-----------|
| CIA (Confidentiality, Integrity, Availability) | Тріада інформаційної безпеки: конфіденційність, цілісність та доступність — три фундаментальні принципи захисту інформації. |
| Конфіденційність | Принцип CIA: інформація доступна лише авторизованим суб'єктам. Забезпечується шифруванням трафіку у VPN. |
| Цілісність | Принцип CIA: гарантія, що дані не змінені несанкціоновано. Забезпечується MAC/HMAC у VPN-протоколах. |
| Доступність | Принцип CIA: система та дані доступні авторизованим користувачам тоді, коли їм це потрібно. |
| VPN (Virtual Private Network) | Віртуальна приватна мережа — технологія, що дозволяє створювати захищений зашифрований тунель поверх публічної мережі. |
| AES (Advanced Encryption Standard) | Симетричний блоковий шифр, прийнятий NIST у 2001 р. Підтримує ключі 128, 192, 256 біт. Стандарт шифрування у більшості VPN. |
| ChaCha20-Poly1305 | Симетричний потоковий шифр із автентифікацією. Швидший за AES на пристроях без апаратного прискорення; використовується у TLS 1.3 та WireGuard. |
| HMAC (Hash-based Message Authentication Code) | Код автентифікації повідомлення на основі хеш-функції та секретного ключа. Забезпечує цілісність та автентифікацію джерела. |
| AEAD (Authenticated Encryption with Associated Data) | Режим шифрування, що поєднує шифрування та автентифікацію. Прикладами є AES-GCM та ChaCha20-Poly1305. |
| GCM (Galois/Counter Mode) | Режим роботи блокового шифру, що реалізує AEAD. Забезпечує одночасно шифрування та автентифікацію. |
| SHA-256 | Криптографічна хеш-функція сімейства SHA-2 з 256-бітним дайджестом. Стандарт цілісності у IPsec та TLS. |
| SHA-3 | Криптографічна хеш-функція на основі конструкції губки Keccak, стандартизована NIST у 2015 р. |
| MD5 | Застарілий хеш-алгоритм з 128-бітним дайджестом. Вважається зламаним, не застосовується у сучасних системах безпеки. |
| MAC (Message Authentication Code) | Код автентифікації повідомлення — криптографічний тег, що підтверджує цілісність та справжність повідомлення. |
| ECB (Electronic Codebook) | Режим блокового шифру, де кожен блок шифрується незалежно. Небезпечний: однакові блоки відкритого тексту дають однакові блоки шифротексту. |
| CBC (Cipher Block Chaining) | Режим блокового шифру, де кожен блок XOR-ується з попереднім шифроблоком перед шифруванням. Вимагає вектора ініціалізації (IV). |
| CTR (Counter Mode) | Потоковий режим блокового шифру, де шифрується лічильник. Дозволяє паралельне шифрування. |
| Site-to-Site VPN | VPN між двома мережами (офісами чи ЦОД), де трафік усієї мережі прозоро тунелюється через шлюзи. |
| Remote Access VPN | VPN для підключення окремих користувачів до корпоративної мережі з будь-якого місця. |
| Інкапсуляція | Процес обгортання пакета одного протоколу в пакет іншого протоколу для передачі через мережу. |
| Тунелювання | Механізм передачі пакетів одного мережевого протоколу всередині пакетів іншого протоколу. |
| DoS/DDoS | Denial of Service / Distributed Denial of Service — атаки на доступність мережевих сервісів шляхом перевантаження трафіком. |
| OSI (Open Systems Interconnection) | Семирівнева модель мережевої взаємодії: фізичний, канальний, мережевий, транспортний, сеансовий, представницький та прикладний рівні. |
| Man-in-the-Middle (MITM) | Атака, при якій зловмисник непомітно перехоплює та може модифікувати трафік між двома сторонами. |
| Rate Limiting | Обмеження швидкості обробки запитів для захисту від DoS-атак та зловживань. |
