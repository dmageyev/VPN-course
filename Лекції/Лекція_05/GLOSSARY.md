# Глосарій — Лекція 05 — Протоколи IPsec

| Термін / Скорочення | Визначення |
|---------------------|-----------|
| IPsec (Internet Protocol Security) | Набір стандартів IETF (RFC 4301) для захисту IP-трафіку на мережевому рівні (L3). Включає протоколи AH, ESP та IKE. |
| SA (Security Association) | Однобічна угода про параметри безпеки між двома вузлами IPsec: алгоритми шифрування, ключі, SPI. SA є однонаправленою. |
| SPI (Security Parameters Index) | 32-бітний індекс у ESP/AH заголовку для ідентифікації Security Association при обробці пакета. |
| SADB (Security Association Database) | База даних активних Security Associations у ядрі ОС. Зберігає параметри шифрування для кожного тунелю. |
| SPD (Security Policy Database) | База даних політик безпеки IPsec. Визначає, який трафік шифрувати, пропускати або відкидати. |
| IKE (Internet Key Exchange) | Протокол обміну ключами для IPsec. IKEv1 (RFC 2409) — застарілий, IKEv2 (RFC 7296) — сучасний стандарт. |
| IKEv2 (Internet Key Exchange v2) | Сучасна версія протоколу IKE (RFC 7296). Підтримує MOBIKE, EAP, пришвидшене повторне підключення та захист від DoS. |
| Phase 1 / IKE SA | Перша фаза IKE: взаємна автентифікація сторін та встановлення захищеного каналу для обміну ключами. |
| Phase 2 / Child SA (IPsec SA) | Друга фаза IKE: узгодження параметрів ESP/AH для захисту трафіку даних. Може бути кілька Child SA в рамках одного IKE SA. |
| xfrm | Підсистема ядра Linux для обробки IPsec. Реалізує SA та SPD на рівні ядра, незалежно від IKE-демона. |
| strongSwan | Відкрита реалізація IKEv2 для Linux/FreeBSD. Найпоширеніший IKE-демон для корпоративних IPsec VPN. |
| libreswan | Відкрита реалізація IKEv1/IKEv2 для Linux, нащадок FreeS/WAN. Широко використовується у Red Hat/Fedora середовищах. |
| Replay Window | Вікно допустимих порядкових номерів пакетів IPsec. Пакети зі Sequence Number поза вікном відкидаються як повторні. |
| ESN (Extended Sequence Numbers) | Розширені порядкові номери IPsec (RFC 4304) — 64-бітні замість 32-бітних. Запобігають переповненню лічильника при високій пропускній здатності. |
| EAP (Extensible Authentication Protocol) | Розширюваний протокол автентифікації, що використовується в IKEv2 для інтеграції з RADIUS, сертифікатами та OTP. |
| RADIUS (Remote Authentication Dial-In User Service) | Протокол AAA (автентифікація, авторизація, облік). Інтегрується з IKEv2 через EAP для централізованої автентифікації користувачів. |
| Policy-Based VPN | IPsec VPN, де шифрування визначається ACL-правилами (політиками). Не вимагає окремого тунельного інтерфейсу. |
| Route-Based VPN | IPsec VPN з окремим тунельним інтерфейсом (vti, xfrmi). Маршрутизація визначає, який трафік іде через тунель. |
| TOTP (Time-based One-Time Password) | Одноразовий пароль на основі часу (RFC 6238). Використовується для MFA у VPN-автентифікації. |
| MTU та MSS Clamping | Налаштування максимального розміру пакета для уникнення фрагментації у IPsec тунелях. Overhead ESP у тунельному режимі — близько 60 байт. |
