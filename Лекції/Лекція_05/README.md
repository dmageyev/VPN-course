# Лекція 05 — Протоколи IPsec

## Мета

Вивчити архітектуру пакету протоколів IPsec, формати кадрів AH та ESP, режими роботи Transport/Tunnel Mode, а також фази та етапи встановлення захищеного з'єднання через IKEv1 та IKEv2.

## План лекції

1. Загальна архітектура IPsec — компоненти та стандарти (RFC)
2. Бази даних SAD та SPD — Security Associations
3. Протокол AH (Authentication Header) — призначення та формат кадру
4. Протокол ESP (Encapsulating Security Payload) — призначення та формат кадру
5. Режими роботи: Transport Mode та Tunnel Mode (формати кадрів)
6. IKEv1 — Фаза 1 (Main Mode, Aggressive Mode) та Фаза 2 (Quick Mode)
7. IKEv2 — спрощене встановлення з'єднання (IKE_SA_INIT, IKE_AUTH, CREATE_CHILD_SA)
8. Криптографічні алгоритми та NAT-Traversal

## Конспект лекції

**IPsec (Internet Protocol Security)** — набір відкритих стандартів для захисту IP-трафіку на мережевому рівні (L3). Забезпечує конфіденційність, цілісність та автентифікацію без зміни застосунків (прозоро для рівня застосунків).

### Security Association (SA)

**SA** — однонаправлений логічний канал між двома вузлами, що визначає всі параметри захисту: алгоритм шифрування, алгоритм автентифікації, ключі, SPI, термін дії, режим. Для двостороннього зв'язку потрібні дві SA. Активні SA зберігаються в **SAD** (Security Association Database). Правила застосування SA визначаються в **SPD** (Security Policy Database) — дії: PROTECT, BYPASS, DISCARD.

### Протоколи AH та ESP

- **AH (IP 51)**: забезпечує автентифікацію та цілісність (включно з незмінними полями IP-заголовка), але **не шифрує** дані. Несумісний з NAT.
- **ESP (IP 50)**: забезпечує шифрування, автентифікацію та цілісність корисного навантаження. Підтримує NAT-T (обгортання в UDP 4500).

### Режими роботи

- **Transport Mode**: зберігається оригінальний IP-заголовок, захищається лише payload. Використовується для Host-to-Host.
- **Tunnel Mode**: весь оригінальний пакет інкапсулюється у новий. IP-адреси приховуються. Використовується у Site-to-Site VPN.

### IKEv1 (RFC 2409)

- **Фаза 1** встановлює захищений IKE SA:
  - *Main Mode*: 6 повідомлень (SA propose → SA accept → KE+Nonce → KE+Nonce → ID+AUTH → ID+AUTH). Повністю захищений.
  - *Aggressive Mode*: 3 повідомлення. Швидший, але вразливий до офлайн-атак на PSK.
- **Фаза 2** (Quick Mode): 3 повідомлення всередині IKE SA. Узгоджує IPsec SA для захисту трафіку. Підтримує PFS через опційний DH-обмін.

### IKEv2 (RFC 7296)

Спрощений протокол. Базове з'єднання потребує лише **4 повідомлення**:
1. **IKE_SA_INIT** (2 пов., відкрито): DH-обмін, nonce, узгодження алгоритмів IKE SA.
2. **IKE_AUTH** (2 пов., зашифровано): автентифікація, встановлення першої Child SA (IPsec SA) з селекторами трафіку.

Додаткові обміни: **CREATE_CHILD_SA** (нові/поновлені IPsec SA з PFS), **INFORMATIONAL** (DELETE, NOTIFY, MOBIKE).
