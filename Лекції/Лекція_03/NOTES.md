# Конспект — Лекція 03 — Мережева архітектура та механізми тунелювання

VPN створює Overlay мережу (віртуальну), яка працює поверх Underlay (фізичної інфраструктури Інтернету). Основним інструментом є інкапсуляція, коли оригінальний пакет стає "корисною нагрузкою" (payload) для нового пакета. Це додає додаткові заголовки, що створює Overhead (накладні витрати) і зменшує корисну смугу пропускання.

Для роботи VPN у операційній системі використовуються віртуальні адаптери:
- TUN (Network Tunnel): Працює на рівні 3 (IP). Найбільш ефективний для маршрутизації.
- TAP (Ethernet Tunnel): Емулює Ethernet-карту (рівень 2). Дозволяє передавати не-IP трафік та створювати мости (bridging).

Критичним аспектом є MTU (Maximum Transmission Unit). Оскільки інкапсуляція збільшує розмір пакета, він може перевищити стандартні 1500 байт Ethernet, що призведе до фрагментації або втрати пакетів. Для виправлення використовується TCP MSS Clamping, що примусово зменшує розмір сегментів даних.

Схема Split-tunnel дозволяє направляти лише корпоративний трафик у VPN, тоді як звичайний інтернет-трафік іде напряму. Full-tunnel захищає весь трафік, але навантажує канал VPN-сервера.
