# Глосарій — Лекція 03 — Мережева архітектура та механізми тунелювання

| Термін / Скорочення | Визначення |
|---------------------|-----------|
| Underlay мережа | Фізична або реальна мережева інфраструктура провайдера: кабелі, маршрутизатори, оптичні магістралі, що забезпечують базову передачу пакетів. |
| Overlay мережа | Віртуальна мережа, побудована поверх Underlay. VPN — приклад Overlay: пакети тунелюються крізь фізичну мережу без знання про неї. |
| VXLAN (Virtual eXtensible LAN) | Протокол інкапсуляції Ethernet у UDP (RFC 7348). Підтримує до 16 мільйонів ізольованих сегментів через 24-бітний VNI. Використовується у хмарних ЦОД. |
| VNI (VXLAN Network Identifier) | 24-бітний ідентифікатор сегменту VXLAN, аналог VLAN ID. Дозволяє мати до 16 млн ізольованих сегментів. |
| VTEP (VXLAN Tunnel Endpoint) | Пристрій або програмний компонент, що інкапсулює та декапсулює VXLAN-пакети. Може бути апаратним або програмним (OVS). |
| Geneve | Протокол тунелювання наступного покоління після VXLAN. Підтримує розширювані метадані у заголовку, застосовується у OpenShift/Kubernetes. |
| SD-WAN (Software-Defined Wide Area Network) | Програмно-визначена глобальна мережа. Централізоване управління, динамічний вибір маршрутів, підтримка кількох каналів WAN з автоматичним перемиканням. |
| ISP (Internet Service Provider) | Інтернет-провайдер — компанія, що надає доступ до Інтернету та є власником Underlay-інфраструктури. |
| MTU (Maximum Transmission Unit) | Максимальний розмір кадру (байт), який може передаватися через мережевий інтерфейс без фрагментації. У VPN зменшується через додаткові заголовки. |
| MSS (Maximum Segment Size) | Максимальний розмір корисних даних у TCP-сегменті. Коригується через MSS Clamping при тунелюванні VPN. |
| DPD (Dead Peer Detection) | Механізм виявлення недоступності VPN-партнера шляхом надсилання контрольних повідомлень. Визначений у RFC 3706. |
| MOBIKE (IKEv2 Mobility and Multihoming) | Розширення IKEv2 (RFC 4555), що дозволяє мобільним клієнтам змінювати IP-адресу без повного переузгодження SA. |
| ECMP (Equal-Cost Multi-Path) | Маршрутизація з кількома рівновартісними маршрутами — балансування навантаження між кількома VPN-тунелями однакової метрики. |
| IPv6 DNS leakage | Витік DNS-запитів через IPv6 у разі, якщо VPN-клієнт не перехоплює IPv6-трафік. Загроза конфіденційності у Dual-Stack середовищах. |
| DSCP (Differentiated Services Code Point) | Поле у IP-заголовку для маркування класу обслуговування (QoS). Копіюється або переписується при тунелюванні у SD-WAN. |
| QoS (Quality of Service) | Набір механізмів управління мережевим трафіком для пріоритизації критичних потоків (голос, відео) над менш важливими. |
| Active/Standby | Схема резервування VPN: один шлюз активний, другий у резерві. При відмові активного — резерв перебирає з'єднання. |
| Active/Active | Схема балансування навантаження VPN: обидва шлюзи одночасно активні. Забезпечує горизонтальне масштабування пропускної здатності. |
| GRE (Generic Routing Encapsulation) | Протокол тунелювання Cisco для інкапсуляції будь-яких пакетів у IP. Заголовок 4 байти. Не шифрує, зазвичай поєднується з IPsec. |
| Dual-Stack VPN Gateway | VPN-шлюз з підтримкою одночасно IPv4 та IPv6, що запобігає витокам трафіку через неохоплений стек. |
