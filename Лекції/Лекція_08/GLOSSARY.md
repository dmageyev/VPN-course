# Глосарій — Лекція 08 — Огляд хмарних VPN-рішень. Моніторинг, діагностика та аналіз інцидентів безпеки

| Термін / Скорочення | Визначення |
|---------------------|-----------|
| AWS VPN (Amazon Web Services VPN) | Хмарний VPN-сервіс AWS: Site-to-Site VPN (IPsec до Virtual Private Gateway) та Client VPN (OpenVPN-сумісний). |
| Azure VPN Gateway | Хмарний VPN-шлюз Microsoft Azure для Site-to-Site та Point-to-Site VPN з підтримкою IKEv2, OpenVPN та SSTP. |
| Google Cloud VPN | Хмарний IPsec VPN Google Cloud. Підтримує HA VPN (99.99% SLA) з двома тунелями для надлишковості. |
| Virtual Private Gateway (VGW) | Компонент AWS VPN: шлюз на стороні AWS у VPC для термінування Site-to-Site IPsec тунелів. |
| Transit Gateway | Хмарний маршрутизатор AWS, що об'єднує кілька VPC та VPN-підключень в єдину хаб-топологію. |
| VPC (Virtual Private Cloud) | Ізольована логічна мережа у хмарному провайдері (AWS, Azure, GCP). Основа хмарної мережевої архітектури. |
| SIEM (Security Information and Event Management) | Система управління подіями та інформацією безпеки. Збирає, нормалізує та аналізує логи з усіх джерел для виявлення інцидентів. |
| SOAR (Security Orchestration, Automation and Response) | Платформа оркестрації, автоматизації та реагування на інциденти безпеки. Автоматизує реакцію на типові загрози. |
| IDS/IPS (Intrusion Detection/Prevention System) | Система виявлення/запобігання вторгненням. Аналізує трафік для виявлення аномалій та відомих атак. |
| DPI (Deep Packet Inspection) | Глибока інспекція пакетів — аналіз змісту пакетів на рівні застосунків. Використовується для фільтрації та виявлення VPN-трафіку. |
| Prometheus | Відкрита система моніторингу та алертингу з моделлю pull-збору метрик. Широко використовується для моніторингу VPN-шлюзів. |
| Grafana | Платформа візуалізації метрик та логів. Використовується разом з Prometheus для дашбордів VPN-статистики. |
| ELK Stack (Elasticsearch, Logstash, Kibana) | Стек для централізованого збору, індексації та візуалізації логів. ELK або OpenSearch використовується для аналізу VPN-логів. |
| VPN Concentrator | Спеціалізований пристрій або ПЗ для термінування великої кількості VPN-з'єднань. Забезпечує масштабованість та централізацію. |
| HA (High Availability) | Висока доступність — архітектурний підхід для мінімізації часу простою через резервування компонентів (Active/Standby або Active/Active). |
| BGP (Border Gateway Protocol) | Протокол динамічної маршрутизації між автономними системами. Використовується у хмарних Site-to-Site VPN для обміну маршрутами. |
| CloudWatch | Сервіс моніторингу та логування AWS. Збирає метрики VPN-з'єднань, трафіку та стан тунелів. |
| Threat Intelligence | Дані про відомі загрози (IP-адреси атакуючих, індикатори компрометації). Використовується у SIEM для кореляції подій. |
| IOC (Indicator of Compromise) | Індикатор компрометації — артефакт (IP, домен, хеш), що свідчить про можливий інцидент безпеки. |
| Anomaly Detection | Виявлення аномалій у мережевому трафіку або поведінці користувачів методами ML або статистичного аналізу для виявлення загроз. |
