# Глосарій — Лекція 06 — SSL/TLS VPN та архітектура OpenVPN

| Термін / Скорочення | Визначення |
|---------------------|-----------|
| SSL/TLS VPN | VPN на основі протоколу SSL/TLS, що працює через стандартні веб-порти (TCP 443). Ефективно проходить через більшість файрволів. |
| OpenVPN | Відкрите VPN-рішення, засноване на бібліотеці OpenSSL. Підтримує UDP/TCP, tun/tap інтерфейси, скриптові хуки та множинні схеми автентифікації. |
| TUN-інтерфейс | Мережевий тунельний інтерфейс рівня 3 (IP). Використовується для маршрутизованих VPN (IP-пакети). |
| TAP-інтерфейс | Мережевий тунельний інтерфейс рівня 2 (Ethernet). Використовується для бриджингових VPN (Ethernet-кадри). |
| TLS Handshake | Процедура узгодження TLS-з'єднання: обмін сертифікатами, вибір шифрнаборів, узгодження ключів через ECDH. |
| Cipher Suite | Набір криптографічних алгоритмів для TLS-з'єднання: алгоритм обміну ключами, шифрування та автентифікації (наприклад, TLS_AES_256_GCM_SHA384). |
| TLS 1.3 | Найсучасніша версія TLS (RFC 8446). Спрощений handshake, обов'язковий PFS, видалено застарілі алгоритми. Стандарт у OpenVPN 2.5+. |
| Control Channel | Керуючий канал OpenVPN на основі TLS. Відповідає за автентифікацію, обмін ключами та управління з'єднанням. |
| Data Channel | Канал даних OpenVPN. Шифрує та передає VPN-трафік. Може використовувати окремі ключі від Control Channel. |
| tls-auth / tls-crypt | Директиви OpenVPN для захисту TLS Handshake за допомогою HMAC або шифрування. tls-crypt надійніший: шифрує та автентифікує control channel. |
| PKCS#8 | Стандарт зберігання приватних ключів. Підтримує шифрування ключа паролем. Використовується в OpenVPN та OpenSSL. |
| OpenSSL | Відкрита бібліотека криптографії. Основа OpenVPN та більшості TLS-реалізацій. Надає утиліти для роботи з сертифікатами та ключами. |
| EasyRSA | Набір скриптів для управління PKI, що постачається з OpenVPN. Спрощує генерацію CA, сертифікатів та ключів. |
| Статичний ключ (OpenVPN) | Режим OpenVPN з попередньо розподіленим спільним ключем (PSK). Простий у налаштуванні, але без PFS та взаємної автентифікації. |
| --push | Директива OpenVPN-сервера для надсилання налаштувань клієнту: маршрути, DNS, параметри підключення. |
| --client-config-dir (CCD) | Директорія файлів індивідуальних налаштувань клієнтів OpenVPN. Дозволяє призначати фіксовані IP, маршрути та параметри на клієнта. |
| Obfsproxy / obfs4 | Плагін маскування трафіку OpenVPN під звичайний HTTPS. Застосовується для обходу DPI у країнах з цензурою. |
| Split Tunneling (OpenVPN) | Налаштування OpenVPN, де лише певні мережі направляються через VPN, а решта — напряму. Реалізується через директиву route. |
| LDAP-автентифікація | Інтеграція OpenVPN з LDAP/Active Directory для централізованої автентифікації користувачів через плагін openvpn-auth-ldap. |
| PAM (Pluggable Authentication Modules) | Система автентифікації Linux. Використовується OpenVPN через плагін openvpn-plugin-auth-pam для інтеграції з системними обліковими записами. |
