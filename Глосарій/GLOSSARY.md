# Глосарій курсу «Технології VPN»

## Терміни та скорочення

| Термін / Скорочення | Визначення | Лекція |
|---------------------|-----------|--------|
| --client-config-dir (CCD) | Директорія файлів індивідуальних налаштувань клієнтів OpenVPN. Дозволяє призначати фіксовані IP, маршрути та параметри на клієнта. | Лекція 06 |
| --push | Директива OpenVPN-сервера для надсилання налаштувань клієнту: маршрути, DNS, параметри підключення. | Лекція 06 |
| ACME (Automatic Certificate Management Environment) | Протокол автоматизації отримання та оновлення сертифікатів TLS, розроблений IETF і використовуваний Let's Encrypt. | Лекція 02 |
| Active/Active | Схема балансування навантаження VPN: обидва шлюзи одночасно активні. Забезпечує горизонтальне масштабування пропускної здатності. | Лекція 03 |
| Active/Standby | Схема резервування VPN: один шлюз активний, другий у резерві. При відмові активного — резерв перебирає з'єднання. | Лекція 03 |
| AEAD (Authenticated Encryption with Associated Data) | Режим шифрування, що поєднує шифрування та автентифікацію. Прикладами є AES-GCM та ChaCha20-Poly1305. | Лекція 01 |
| AES (Advanced Encryption Standard) | Симетричний блоковий шифр, прийнятий NIST у 2001 р. Підтримує ключі 128, 192, 256 біт. Стандарт шифрування у більшості VPN. | Лекція 01 |
| AH (Authentication Header) | Протокол IPsec (RFC 4302), що забезпечує тільки автентифікацію та цілісність без шифрування. Протокол IP 51. | Лекція 04 |
| AllowedIPs | Параметр WireGuard peer: список IP-діапазонів, які маршрутизуються через цього peer. Основа Cryptokey Routing. | Лекція 07 |
| Anomaly Detection | Виявлення аномалій у мережевому трафіку або поведінці користувачів методами ML або статистичного аналізу для виявлення загроз. | Лекція 08 |
| AWS VPN (Amazon Web Services VPN) | Хмарний VPN-сервіс AWS: Site-to-Site VPN (IPsec до Virtual Private Gateway) та Client VPN (OpenVPN-сумісний). | Лекція 08 |
| Azure VPN Gateway | Хмарний VPN-шлюз Microsoft Azure для Site-to-Site та Point-to-Site VPN з підтримкою IKEv2, OpenVPN та SSTP. | Лекція 08 |
| BeyondCorp | Модель Zero Trust від Google, що реалізує ZTNA для всіх корпоративних застосунків незалежно від мережевого розташування. | Лекція 07 |
| BGP (Border Gateway Protocol) | Протокол динамічної маршрутизації між автономними системами. Використовується у хмарних Site-to-Site VPN для обміну маршрутами. | Лекція 08 |
| BLAKE2s | Криптографічна хеш-функція, що використовується у WireGuard для хешування та KDF. Швидша за SHA-2 при порівняній безпеці. | Лекція 07 |
| CA (Certificate Authority) | Центр сертифікації — довірена організація, що видає та підписує цифрові сертифікати X.509. | Лекція 02 |
| CBC (Cipher Block Chaining) | Режим блокового шифру, де кожен блок XOR-ується з попереднім шифроблоком перед шифруванням. Вимагає вектора ініціалізації (IV). | Лекція 01 |
| ChaCha20-Poly1305 | Симетричний потоковий шифр із автентифікацією. Швидший за AES на пристроях без апаратного прискорення; використовується у TLS 1.3 та WireGuard. | Лекція 01 |
| CIA (Confidentiality, Integrity, Availability) | Тріада інформаційної безпеки: конфіденційність, цілісність та доступність — три фундаментальні принципи захисту інформації. | Лекція 01 |
| Cipher Suite | Набір криптографічних алгоритмів для TLS-з'єднання: алгоритм обміну ключами, шифрування та автентифікації (наприклад, TLS_AES_256_GCM_SHA384). | Лекція 06 |
| CloudWatch | Сервіс моніторингу та логування AWS. Збирає метрики VPN-з'єднань, трафіку та стан тунелів. | Лекція 08 |
| Continuous Authentication | Безперервна автентифікація у Zero Trust: ідентичність підтверджується повторно при кожному запиті або через короткі інтервали. | Лекція 07 |
| Control Channel | Керуючий канал OpenVPN на основі TLS. Відповідає за автентифікацію, обмін ключами та управління з'єднанням. | Лекція 06 |
| CRL (Certificate Revocation List) | Список відкликаних сертифікатів — список серійних номерів сертифікатів, які CA визнав недійсними до закінчення терміну дії. | Лекція 02 |
| Cryptokey Routing | Концепція WireGuard: кожному публічному ключу відповідають дозволені IP-діапазони. Спрощує маршрутизацію та авторизацію. | Лекція 07 |
| CSR (Certificate Signing Request) | Запит на підписання сертифіката — файл, що містить відкритий ключ та інформацію про суб'єкт, надсилається в CA для отримання сертифіката. | Лекція 02 |
| CTR (Counter Mode) | Потоковий режим блокового шифру, де шифрується лічильник. Дозволяє паралельне шифрування. | Лекція 01 |
| Curve25519 | Еліптична крива, оптимізована для ECDH. Використовується у WireGuard та сучасних реалізаціях TLS завдяки швидкості та безпеці. | Лекція 02 |
| Data Channel | Канал даних OpenVPN. Шифрує та передає VPN-трафік. Може використовувати окремі ключі від Control Channel. | Лекція 06 |
| DERP (Designated Encrypted Relay for Packets) | Протокол та сервери-ретранслятори Tailscale для передачі WireGuard-трафіку через NAT у разі, коли пряме підключення неможливе. | Лекція 07 |
| Diffie-Hellman (DH) | Протокол обміну ключами у відкритому каналі без попереднього спільного секрету. Основа більшості VPN-протоколів для узгодження ключів. | Лекція 02 |
| DoS/DDoS | Denial of Service / Distributed Denial of Service — атаки на доступність мережевих сервісів шляхом перевантаження трафіком. | Лекція 01 |
| DPD (Dead Peer Detection) | Механізм виявлення недоступності VPN-партнера шляхом надсилання контрольних повідомлень. Визначений у RFC 3706. | Лекція 03 |
| DPI (Deep Packet Inspection) | Глибока інспекція пакетів — аналіз змісту пакетів на рівні застосунків. Використовується для фільтрації та виявлення VPN-трафіку. | Лекція 08 |
| DSCP (Differentiated Services Code Point) | Поле у IP-заголовку для маркування класу обслуговування (QoS). Копіюється або переписується при тунелюванні у SD-WAN. | Лекція 03 |
| Dual-Stack VPN Gateway | VPN-шлюз з підтримкою одночасно IPv4 та IPv6, що запобігає витокам трафіку через неохоплений стек. | Лекція 03 |
| EAP (Extensible Authentication Protocol) | Розширюваний протокол автентифікації, що використовується в IKEv2 для інтеграції з RADIUS, сертифікатами та OTP. | Лекція 05 |
| EasyRSA | Набір скриптів для управління PKI, що постачається з OpenVPN. Спрощує генерацію CA, сертифікатів та ключів. | Лекція 06 |
| ECB (Electronic Codebook) | Режим блокового шифру, де кожен блок шифрується незалежно. Небезпечний: однакові блоки відкритого тексту дають однакові блоки шифротексту. | Лекція 01 |
| ECC (Elliptic Curve Cryptography) | Асиметрична криптографія на основі еліптичних кривих. Забезпечує еквівалентну стійкість при значно менших розмірах ключів порівняно з RSA. | Лекція 02 |
| ECDH (Elliptic Curve Diffie-Hellman) | Протокол обміну ключами на основі еліптичних кривих. Основа для Perfect Forward Secrecy у TLS та IKE. | Лекція 02 |
| ECDSA (Elliptic Curve Digital Signature Algorithm) | Алгоритм цифрового підпису на основі еліптичних кривих. Використовується у TLS-сертифікатах та SSH-ключах. | Лекція 02 |
| ECMP (Equal-Cost Multi-Path) | Маршрутизація з кількома рівновартісними маршрутами — балансування навантаження між кількома VPN-тунелями однакової метрики. | Лекція 03 |
| Ed25519 | Схема цифрового підпису на основі кривої Edwards25519. Використовується у SSH, WireGuard та PGP як стандарт сучасного підпису. | Лекція 02 |
| ELK Stack (Elasticsearch, Logstash, Kibana) | Стек для централізованого збору, індексації та візуалізації логів. ELK або OpenSearch використовується для аналізу VPN-логів. | Лекція 08 |
| ESN (Extended Sequence Numbers) | Розширені порядкові номери IPsec (RFC 4304) — 64-бітні замість 32-бітних. Запобігають переповненню лічильника при високій пропускній здатності. | Лекція 05 |
| ESP (Encapsulating Security Payload) | Протокол IPsec (RFC 4303), що забезпечує шифрування, автентифікацію та захист від replay-атак. Протокол IP 50. | Лекція 04 |
| Full Tunneling | Режим VPN, де весь трафік клієнта направляється через VPN-тунель. Забезпечує максимальний контроль. | Лекція 04 |
| GCM (Galois/Counter Mode) | Режим роботи блокового шифру, що реалізує AEAD. Забезпечує одночасно шифрування та автентифікацію. | Лекція 01 |
| Geneve | Протокол тунелювання наступного покоління після VXLAN. Підтримує розширювані метадані у заголовку, застосовується у OpenShift/Kubernetes. | Лекція 03 |
| Google Cloud VPN | Хмарний IPsec VPN Google Cloud. Підтримує HA VPN (99.99% SLA) з двома тунелями для надлишковості. | Лекція 08 |
| Grafana | Платформа візуалізації метрик та логів. Використовується разом з Prometheus для дашбордів VPN-статистики. | Лекція 08 |
| GRE (Generic Routing Encapsulation) | Протокол тунелювання Cisco для інкапсуляції будь-яких пакетів у IP. Заголовок 4 байти. Не шифрує, зазвичай поєднується з IPsec. | Лекція 03 |
| HA (High Availability) | Висока доступність — архітектурний підхід для мінімізації часу простою через резервування компонентів (Active/Standby або Active/Active). | Лекція 08 |
| Headscale | Відкрита self-hosted альтернатива control plane Tailscale для управління WireGuard mesh-мережею. | Лекція 07 |
| HMAC (Hash-based Message Authentication Code) | Код автентифікації повідомлення на основі хеш-функції та секретного ключа. Забезпечує цілісність та автентифікацію джерела. | Лекція 01 |
| Identity-Aware Proxy (IAP) | Проксі, що перевіряє ідентичність та контекст перед наданням доступу до ресурсу. Ключовий компонент ZTNA-архітектур. | Лекція 07 |
| IDS/IPS (Intrusion Detection/Prevention System) | Система виявлення/запобігання вторгненням. Аналізує трафік для виявлення аномалій та відомих атак. | Лекція 08 |
| IKE (Internet Key Exchange) | Протокол обміну ключами для IPsec. IKEv1 (RFC 2409) — застарілий, IKEv2 (RFC 7296) — сучасний стандарт. | Лекція 05 |
| IKEv2 (Internet Key Exchange v2) | Сучасна версія протоколу IKE (RFC 7296). Підтримує MOBIKE, EAP, пришвидшене повторне підключення та захист від DoS. | Лекція 05 |
| IKEv2 (Internet Key Exchange version 2) | Протокол обміну ключами для IPsec (RFC 7296). Підтримує MOBIKE, швидке перепідключення, EAP-автентифікацію. | Лекція 04 |
| Intermediate CA | Проміжний центр сертифікації — підпорядкований Root CA. Видає кінцеві сертифікати, ізолюючи Root CA від прямих операцій. | Лекція 02 |
| IOC (Indicator of Compromise) | Індикатор компрометації — артефакт (IP, домен, хеш), що свідчить про можливий інцидент безпеки. | Лекція 08 |
| IPsec (Internet Protocol Security) | Набір стандартів IETF (RFC 4301) для захисту IP-трафіку на мережевому рівні (L3). Включає протоколи AH, ESP та IKE. | Лекція 05 |
| IPv6 DNS leakage | Витік DNS-запитів через IPv6 у разі, якщо VPN-клієнт не перехоплює IPv6-трафік. Загроза конфіденційності у Dual-Stack середовищах. | Лекція 03 |
| ISP (Internet Service Provider) | Інтернет-провайдер — компанія, що надає доступ до Інтернету та є власником Underlay-інфраструктури. | Лекція 03 |
| Key rotation (WireGuard) | Регулярна ротація попередньо розподілених ключів (PSK) у WireGuard для підвищення PFS та захисту від майбутнього квантового зламу. | Лекція 07 |
| L2TP (Layer 2 Tunneling Protocol) | Протокол тунелювання канального рівня (RFC 2661). Не шифрує самостійно; зазвичай використовується разом з IPsec (L2TP/IPsec). | Лекція 04 |
| L2TP/IPsec | Комбінація L2TP для тунелювання та IPsec для шифрування. Поширений у корпоративних VPN, вбудований у Windows, macOS, iOS, Android. | Лекція 04 |
| LDAP-автентифікація | Інтеграція OpenVPN з LDAP/Active Directory для централізованої автентифікації користувачів через плагін openvpn-auth-ldap. | Лекція 06 |
| Let's Encrypt | Безкоштовний автоматизований CA з підтримкою протоколу ACME для автоматичного отримання та оновлення TLS-сертифікатів. | Лекція 02 |
| libreswan | Відкрита реалізація IKEv1/IKEv2 для Linux, нащадок FreeS/WAN. Широко використовується у Red Hat/Fedora середовищах. | Лекція 05 |
| MAC (Message Authentication Code) | Код автентифікації повідомлення — криптографічний тег, що підтверджує цілісність та справжність повідомлення. | Лекція 01 |
| MagicDNS | Функція Tailscale: автоматичне DNS-розв'язання імен вузлів у mesh-мережі без ручного налаштування DNS. | Лекція 07 |
| Man-in-the-Middle (MITM) | Атака, при якій зловмисник непомітно перехоплює та може модифікувати трафік між двома сторонами. | Лекція 01 |
| MD5 | Застарілий хеш-алгоритм з 128-бітним дайджестом. Вважається зламаним, не застосовується у сучасних системах безпеки. | Лекція 01 |
| MOBIKE (IKEv2 Mobility and Multihoming) | Розширення IKEv2 (RFC 4555), що дозволяє мобільним клієнтам змінювати IP-адресу без повного переузгодження SA. | Лекція 03 |
| MS-CHAPv2 | Microsoft Challenge Handshake Authentication Protocol v2 — протокол автентифікації. Вразливий до офлайн-атак перебором; не рекомендується без захисного тунеля. | Лекція 04 |
| MSS (Maximum Segment Size) | Максимальний розмір корисних даних у TCP-сегменті. Коригується через MSS Clamping при тунелюванні VPN. | Лекція 03 |
| MTU (Maximum Transmission Unit) | Максимальний розмір кадру (байт), який може передаватися через мережевий інтерфейс без фрагментації. У VPN зменшується через додаткові заголовки. | Лекція 03 |
| MTU та MSS Clamping | Налаштування максимального розміру пакета для уникнення фрагментації у IPsec тунелях. Overhead ESP у тунельному режимі — близько 60 байт. | Лекція 05 |
| NAT Traversal (NAT-T) | Механізм для роботи IPsec через NAT шляхом інкапсуляції ESP у UDP (порт 4500). | Лекція 04 |
| Noise Protocol Framework | Криптографічний фреймворк, на якому побудований WireGuard. Визначає безпечне рукостискання та управління ключами. | Лекція 07 |
| Obfsproxy / obfs4 | Плагін маскування трафіку OpenVPN під звичайний HTTPS. Застосовується для обходу DPI у країнах з цензурою. | Лекція 06 |
| OCSP (Online Certificate Status Protocol) | Протокол онлайн-перевірки статусу сертифіката в реальному часі — альтернатива CRL. | Лекція 02 |
| OpenSSL | Відкрита бібліотека криптографії. Основа OpenVPN та більшості TLS-реалізацій. Надає утиліти для роботи з сертифікатами та ключами. | Лекція 06 |
| OpenVPN | VPN-рішення з відкритим кодом, що використовує TLS для управління та шифрування трафіку. Підтримує UDP/TCP, порт 1194 за замовчуванням. | Лекція 04 |
| OSI (Open Systems Interconnection) | Семирівнева модель мережевої взаємодії: фізичний, канальний, мережевий, транспортний, сеансовий, представницький та прикладний рівні. | Лекція 01 |
| Overlay мережа | Віртуальна мережа, побудована поверх Underlay. VPN — приклад Overlay: пакети тунелюються крізь фізичну мережу без знання про неї. | Лекція 03 |
| PAM (Pluggable Authentication Modules) | Система автентифікації Linux. Використовується OpenVPN через плагін openvpn-plugin-auth-pam для інтеграції з системними обліковими записами. | Лекція 06 |
| PEM (Privacy-Enhanced Mail) | Текстовий формат кодування сертифікатів та ключів у Base64 між заголовками BEGIN/END. Найпоширеніший формат у Linux/OpenSSL. | Лекція 02 |
| PFS (Perfect Forward Secrecy) | Властивість протоколу: компрометація довгострокового ключа не розкриває сесійних ключів минулих сеансів. Реалізується через ECDH. | Лекція 02 |

| Phase 2 / Child SA (IPsec SA) | Друга фаза IKE: узгодження параметрів ESP/AH для захисту трафіку даних. Може бути кілька Child SA в рамках одного IKE SA. | Лекція 05 |

| PPTP (Point-to-Point Tunneling Protocol) | Застарілий VPN-протокол Microsoft (RFC 2637) на основі PPP. Вважається небезпечним через вразливості MS-CHAPv2 та MPPE. | Лекція 04 |
| Preshared Key (PSK) | Додатковий симетричний ключ у WireGuard поверх ECDH. Підвищує квантову стійкість та забезпечує додатковий шар безпеки. | Лекція 07 |
| Prometheus | Відкрита система моніторингу та алертингу з моделлю pull-збору метрик. Широко використовується для моніторингу VPN-шлюзів. | Лекція 08 |
| QoS (Quality of Service) | Набір механізмів управління мережевим трафіком для пріоритизації критичних потоків (голос, відео) над менш важливими. | Лекція 03 |
| RADIUS (Remote Authentication Dial-In User Service) | Протокол AAA (автентифікація, авторизація, облік). Інтегрується з IKEv2 через EAP для централізованої автентифікації користувачів. | Лекція 05 |
| Rate Limiting | Обмеження швидкості обробки запитів для захисту від DoS-атак та зловживань. | Лекція 01 |
| Remote Access VPN | VPN для підключення окремих користувачів до корпоративної мережі з будь-якого місця. | Лекція 01 |
| Replay Window | Вікно допустимих порядкових номерів пакетів IPsec. Пакети зі Sequence Number поза вікном відкидаються як повторні. | Лекція 05 |
| Roaming (WireGuard) | Здатність WireGuard-клієнта змінювати IP-адресу без переузгодження тунелю. Реалізовано через відстеження останньої активної endpoint-адреси. | Лекція 07 |
| Root CA | Кореневий центр сертифікації — верхній рівень ієрархії PKI, чий сертифікат є самопідписаним та попередньо встановлений у довірені сховища. | Лекція 02 |
| Route-Based VPN | IPsec VPN з окремим тунельним інтерфейсом (vti, xfrmi). Маршрутизація визначає, який трафік іде через тунель. | Лекція 05 |
| RSA (Rivest–Shamir–Adleman) | Асиметричний алгоритм шифрування та електронного підпису, що базується на складності факторизації великих чисел. Розміри ключів: 2048–4096 біт. | Лекція 02 |

| SHA-256 | Криптографічна хеш-функція сімейства SHA-2 з 256-бітним дайджестом. Стандарт цілісності у IPsec та TLS. | Лекція 01 |

| Site-to-Site VPN | VPN між двома мережами (офісами чи ЦОД), де трафік усієї мережі прозоро тунелюється через шлюзи. | Лекція 01 |
